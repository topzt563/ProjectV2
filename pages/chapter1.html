พื้นฐาน Version Control

บทที่ 1 – ทำความรู้จัก Git

ทำความเข้าใจว่า Git คืออะไร ทำงานอย่างไร และทำไมจึงเป็นเครื่องมือจำเป็นสำหรับนักพัฒนาทุกคน

▶ วิดีโอแนะนำ – Git คืออะไร? (Git in 100 Seconds)

Version Control System คืออะไร?

Version Control System (VCS) คือระบบที่ช่วยบันทึกการเปลี่ยนแปลงของไฟล์เมื่อเวลาผ่านไป ทำให้คุณสามารถย้อนกลับไปดูเวอร์ชันก่อนหน้าได้เสมอ

ลองนึกภาพว่าคุณกำลังเขียนโค้ดอยู่ แล้วเกิดความผิดพลาดขึ้น โดยไม่มี VCS คุณอาจต้องเขียนโค้ดใหม่ทั้งหมด แต่ถ้ามี Git คุณแค่พิมพ์คำสั่งเดียวเพื่อย้อนกลับ

Git คืออะไร?

Git คือ Distributed Version Control System (DVCS) ที่พัฒนาโดย Linus Torvalds ผู้สร้าง Linux kernel ในปี 2005 ปัจจุบัน Git กลายเป็นมาตรฐานอุตสาหกรรมที่นักพัฒนาทั่วโลกใช้งาน

💡 ข้อดีของ Git
  • ✅ ทำงานได้แบบ Offline – ไม่ต้องเชื่อมต่ออินเทอร์เน็ตตลอดเวลา
  • ✅ เร็วมาก – ออกแบบมาให้ประมวลผลได้รวดเร็ว
  • ✅ ฟรี & Open Source – ใช้ได้ทุกแพลตฟอร์ม
  • ✅ รองรับการ ทำงานเป็นทีม ได้ดีเยี่ยม

ความแตกต่าง Git vs GitHub

🔧
Git
เครื่องมือ (Software) ที่ติดตั้งในเครื่องของคุณ ใช้ติดตามการเปลี่ยนแปลงของไฟล์บนเครื่อง
🐙
GitHub
เว็บไซต์ / Cloud Platform สำหรับเก็บโค้ดออนไลน์ ทำงานร่วมกับทีม และแชร์โปรเจกต์

แนวคิดหลักของ Git

Repository
โฟลเดอร์โปรเจกต์ที่ Git ดูแลอยู่ มีโฟลเดอร์ .git ซ่อนอยู่ข้างใน
Commit
การบันทึก snapshot ของโค้ด ณ ขณะนั้น เหมือนการ save เกม
Branch
สายการพัฒนาแยกออกมา ทำให้ทำงานหลายอย่างพร้อมกันได้
Merge
การรวม Branch ต่างๆ เข้าด้วยกัน
📌 สรุปบทที่ 1
Git คือระบบควบคุมเวอร์ชัน ช่วยติดตามการเปลี่ยนแปลง ย้อนกลับโค้ด และทำงานร่วมกับทีม GitHub คือแพลตฟอร์มออนไลน์ที่ใช้จัดเก็บ Git repository บนคลาวด์
