บทที่ 3 – คำสั่งพื้นฐาน Git

เรียนรู้คำสั่ง Git ที่ใช้บ่อยที่สุด ตั้งแต่สร้าง repo ไปจนถึง push โค้ดขึ้น GitHub

▶ วิดีโอสาธิต – คำสั่ง Git พื้นฐานที่ต้องรู้

ขั้นตอนการทำงานกับ Git (Workflow)

Working Directory
→
Staging Area
→
Local Repo
→
Remote (GitHub)

git init – เริ่มต้น Repository

ใช้คำสั่งนี้เพื่อเปลี่ยนโฟลเดอร์ปกติให้กลายเป็น Git repository

bash
# สร้างโฟลเดอร์ใหม่แล้ว init mkdir my-project cd my-project git init Initialized empty Git repository in /my-project/.git/

git status – ดูสถานะไฟล์

คำสั่งที่ใช้บ่อยที่สุด ตรวจสอบว่าไฟล์ไหนถูกแก้ไข ไฟล์ไหนพร้อม commit

bash
git status On branch main Untracked files: index.html (use "git add" to track)

git add – เพิ่มไฟล์ไปยัง Staging Area

bash
# เพิ่มไฟล์เดียว git add index.html # เพิ่มทุกไฟล์ที่เปลี่ยนแปลง git add . # เพิ่มเฉพาะไฟล์ .html ทั้งหมด git add *.html

git commit – บันทึกการเปลี่ยนแปลง

📝 แนวทางการเขียน Commit Message ที่ดี
ใช้ภาษาที่ชัดเจน เช่น "Add login form", "Fix navigation bug", "Update README" ไม่ควรเขียนแบบคลุมเครือ เช่น "fix" หรือ "update"
bash
# commit พร้อมข้อความ git commit -m "Add homepage HTML structure" # commit และ add ไฟล์ที่ติดตามอยู่แล้วพร้อมกัน git commit -am "Update styles and fix layout" [main 3a8f9c1] Add homepage HTML structure 1 file changed, 42 insertions(+)

git log – ดูประวัติ Commit

bash
# ดู log แบบละเอียด git log # ดู log แบบสั้น (แนะนำ) git log --oneline 3a8f9c1 Add homepage HTML structure b7d2e4f Initial commit

git push & git pull – ส่ง/รับโค้ดกับ Remote

bash
# push โค้ดขึ้น GitHub (ครั้งแรก) git push -u origin main # push ครั้งถัดไป git push # pull โค้ดล่าสุดจาก GitHub git pull Already up to date.

ตารางคำสั่งอ้างอิงด่วน

คำสั่ง ความหมาย
git initสร้าง Git repository ใหม่
git statusดูสถานะไฟล์ใน working directory
git add <file>เพิ่มไฟล์ไปยัง Staging Area
git add .เพิ่มทุกไฟล์ที่เปลี่ยนแปลงไปยัง Staging
git commit -m "msg"บันทึกการเปลี่ยนแปลงพร้อมข้อความ
git log --onelineดูประวัติ commit แบบย่อ
git pushส่งโค้ดขึ้น Remote repository
git pullดึงโค้ดล่าสุดจาก Remote
git clone <url>คัดลอก repository จาก Remote มาไว้ในเครื่อง
git diffดูความแตกต่างของไฟล์ที่ยังไม่ได้ stage
git restore <file>ยกเลิกการแก้ไขไฟล์ใน working directory
git reset HEAD <file>นำไฟล์ออกจาก Staging Area
