บทที่ 5 – Branch & Merge

การแตก Branch เพื่อพัฒนาฟีเจอร์แยก และรวมโค้ดกลับด้วย Merge อย่างถูกวิธี

▶ วิดีโอสาธิต – Git Branching และ Merging

Branch คืออะไร?

Branch คือสายการพัฒนาที่แยกออกมาจาก main สามารถทำงานหลายฟีเจอร์พร้อมกันได้โดยไม่กระทบกัน เมื่อเสร็จแล้วจึงนำมารวมกัน (Merge)

BRANCH DIAGRAM
main ●───●───●──────────────────● (merge) └──●───●───● feature/login └──●───● bugfix/nav

คำสั่งจัดการ Branch

bash – การสร้างและสลับ Branch
# ดู branch ทั้งหมด (* = branch ปัจจุบัน) git branch * main feature/login # สร้าง branch ใหม่ git branch feature/login # สลับไปยัง branch ที่ต้องการ git checkout feature/login # สร้างและสลับไป branch ใหม่ในคำสั่งเดียว (แนะนำ) git checkout -b feature/login # (Git เวอร์ชันใหม่) ใช้ switch แทน checkout git switch -c feature/login

Merge – รวม Branch

bash – การ Merge branch
# 1. สลับกลับไปที่ main ก่อน git checkout main # 2. Merge feature branch เข้ามา git merge feature/login Updating 3a8f9c1..7b2e4d8 Fast-forward login.html | 45 ++++++++++++ # 3. ลบ branch ที่ merge แล้ว (ถ้าไม่ต้องการแล้ว) git branch -d feature/login

แก้ไข Merge Conflict

Conflict เกิดขึ้นเมื่อ 2 branch แก้ไขส่วนเดียวกันของไฟล์ Git จะทำเครื่องหมายไว้ในไฟล์เพื่อให้แก้ไขด้วยตัวเอง

html – ตัวอย่าง Conflict ที่ Git สร้างขึ้น
<<<<<<< HEAD <h1>สวัสดี GitLearn</h1> ======= <h1>ยินดีต้อนรับสู่ GitLearn</h1> >>>>>>> feature/login
  1. เปิดไฟล์ที่มี conflict
    Git จะบอกว่าไฟล์ไหน conflict ผ่าน git status
  2. แก้ไขโค้ดในไฟล์
    ลบ marker (<<<, ===, >>>) ออก แล้วเลือกโค้ดที่ต้องการหรือรวมทั้งสองเข้าด้วยกัน
  3. git add ไฟล์ที่แก้แล้ว
    git add index.html
  4. git commit เพื่อสรุปการ merge
    git commit -m "Resolve merge conflict"
💡 Best Practice
  • ✅ ตั้งชื่อ branch ให้สื่อความหมาย เช่น feature/login, bugfix/header
  • ✅ Merge บ่อยๆ เพื่อลด conflict
  • ✅ ใช้ git pull ดึงโค้ดใหม่ก่อนเริ่มทำงานทุกครั้ง
  • ✅ ลบ branch ที่ merge แล้วเพื่อความเรียบร้อย
🎉

ยินดีด้วย! คุณเรียนจบแล้ว
