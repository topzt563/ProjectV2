GitHub Repository Pull Request

บทที่ 4 – GitHub เบื้องต้น

เรียนรู้การสร้าง Repository, Clone โปรเจกต์, Fork และเปิด Pull Request เพื่อร่วมพัฒนากับทีม

▶ วิดีโอสาธิต – GitHub สำหรับมือใหม่

สร้าง Repository บน GitHub

  1. คลิก "New" หรือ "+" แล้วเลือก "New repository"
    ที่มุมขวาบนของ GitHub.com
  2. ตั้งชื่อ Repository
    ใส่ชื่อโปรเจกต์ เช่น my-first-repo เลือก Public หรือ Private
  3. เลือกตัวเลือกเพิ่มเติม
    ติ๊ก "Add a README file" เพื่อสร้างไฟล์ README อัตโนมัติ
  4. คลิก "Create repository"
    Repository พร้อมใช้งานแล้ว!

เชื่อมต่อ Local กับ GitHub (Remote)

bash – เชื่อม local repo กับ GitHub
# เพิ่ม remote origin (แทนที่ URL ด้วย URL repo ของคุณ) git remote add origin https://github.com/username/my-first-repo.git # ตรวจสอบ remote ที่ตั้งค่าไว้ git remote -v origin https://github.com/username/my-first-repo.git (fetch) origin https://github.com/username/my-first-repo.git (push) # push โค้ดขึ้น GitHub ครั้งแรก git push -u origin main

Clone Repository

การ Clone คือการดาวน์โหลด repository จาก GitHub มาไว้ในเครื่องพร้อมประวัติ commit ทั้งหมด

bash
git clone https://github.com/username/my-first-repo.git # clone แล้วตั้งชื่อโฟลเดอร์เอง git clone https://github.com/username/my-first-repo.git my-folder

Fork & Pull Request

Fork คือการ copy repository ของคนอื่นมาเป็นของตัวเอง เพื่อแก้ไขโดยไม่กระทบต้นฉบับ

Pull Request (PR) คือการขอรวมโค้ดของเราเข้าไปใน repository ต้นทาง

🍴 ขั้นตอน Fork
  1. 1. ไปที่ repo ที่ต้องการ
  2. 2. คลิกปุ่ม "Fork" มุมขวาบน
  3. 3. เลือก account ของคุณ
  4. 4. Clone fork มาทำงานในเครื่อง
🔀 ขั้นตอน Pull Request
  1. 1. Push branch ขึ้น GitHub ของคุณ
  2. 2. ไปที่ repo ต้นทาง
  3. 3. คลิก "Compare & pull request"
  4. 4. เขียนคำอธิบาย แล้วคลิก "Create PR"
⚠️ ข้อควรระวัง
อย่า push ตรงไปที่ main branch ในโปรเจกต์ทีม ควรสร้าง branch แยก แล้วทำ Pull Request เพื่อให้ทีมร่วม review โค้ดก่อนเสมอ
